## 關鍵 API 行為發現

1. **Topics API** 建立時必須帶 `category_labels` 欄位，否則 422
2. **Prompt clone** 回傳 `{prompt: {id, ...}, message}`，id 在 prompt 子物件
3. **Active prompt** 結構為 `{prompt: {id, ...}, is_default}`
4. **自訂 prompt 上限 3 個**，超過回 400 `自訂人物設定最多 3 個`
5. **TTS characters** 為字串陣列 `["healthy2", "healthy3"]`
6. **httpx.Client.delete()** 不支援 `json` 參數，需用 `request("DELETE", url, json=...)`
7. **Runtime-settings** 端點不存在（404），字數限制可能在前端或 prompt 設定中
8. **previous_session_id** — 帶此參數建立新 session 後，舊 session 仍可使用（回 200）
9. **CSV 含 img 欄位** — 自動拆分為多檔，以 IMG_ID 分組，大小寫皆可（IMG_/img_/Img_）
10. **刪除最後一個文件** → 該 topic 自動從列表移除
11. **TTS GET 端點** — 回傳 audio/wav 二進位，非 JSON；200 + audio content-type = ready
12. **知識 Gemini 同步** — 上傳後約 10 秒可透過 chat citations 檢索到
13. **對話效能** — 8 輪連續對話退化僅 3%，平均回應 5.4s
14. **語系隔離** — zh/en 文件列表獨立，上傳時 language 參數決定歸屬

## 風險評估

1. **知識庫檢索準確度**（L-01, L-03, K-03）：Citation 引用錯誤、代詞解析失敗、新科別未即時同步，直接影響衛教資訊可信度 → 最需優先處理
2. **違法/敏感內容統一處理缺失**（J-06）：目前多為個案式拒答，缺乏大範圍策略與關懷語氣規範
3. **UI 語系切換遺漏**（H-01）：英文模式下多處介面殘留中文，影響國際化體驗
4. **偶發庫外拒答失效**（J-01）：同問題出現不同回應（一次答一次拒），需穩定化拒答策略
